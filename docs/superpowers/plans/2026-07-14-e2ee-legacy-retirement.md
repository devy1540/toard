# E2EE Legacy Key Retirement Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 전체 `server_v1` 0건, 백업 보존 만료, 운영자 확인, 서버 KEK 제거를 증명 가능한 상태로 관리하고 잘못된 키 제거 배포를 차단한다.

**Architecture:** PostgreSQL singleton과 append-only 이벤트로 전역 폐기 상태를 보존하고, 순수 상태 계산 함수가 DB 관측값·환경설정·시각을 관리자 상태로 변환한다. 관리자 API/UI는 확인 동작만 제공하며 실제 백업과 Secret Store는 조작하지 않는다. `/api/ready`는 legacy 행이 남았는데 KEK가 없는 배포만 fail-closed한다.

**Tech Stack:** Next.js 15 App Router, TypeScript, PostgreSQL, node:test, next-intl, shadcn/ui

## Global Constraints

- 프로덕션 DB를 직접 수정하지 않고 migration 파일로만 스키마를 변경한다.
- KEK 값, 암호문, 백업 경로를 API·UI·로그·감사 이벤트에 저장하거나 출력하지 않는다.
- 외부 백업과 Secret Store를 앱에서 자동 삭제하지 않는다.
- `TOARD_LEGACY_BACKUP_RETENTION_DAYS`는 0~3650 정수만 허용하고 미설정은 안전한 대기 상태다.
- readiness는 `server_v1 > 0`이면서 유효한 KEK가 없을 때만 새로 실패한다.
