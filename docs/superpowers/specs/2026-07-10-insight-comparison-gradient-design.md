# 인사이트 비교 그래프 그라데이션 설계

## 배경

인사이트의 현재·이전 기간 추이는 두 개의 선으로만 표현한다. 현재 기간은 브랜드 색상의 실선, 이전 기간은 회색 점선이어서 비교는 명확하지만, `내 사용량`과 `전체 현황`의 사용량 그래프가 사용하는 그라데이션 면 스타일과 시각적 문법이 다르다.

현재 기간에만 옅은 그라데이션 면을 추가해 대시보드 전체의 통일감을 높인다. 이전 기간은 점선으로 유지해 두 기간의 겹침과 교차 지점을 계속 명확하게 읽을 수 있게 한다.

## 목표

- 현재 기간 실선 아래에 위에서 아래로 옅어지는 면 그라데이션을 표시한다.
- 기존 대시보드의 색상 토큰과 그라데이션 방향을 재사용한다.
- 이전 기간은 회색 점선만 표시해 비교 가독성을 유지한다.
- 비용과 토큰 지표 모두 같은 시각 규칙을 사용한다.
- 기존 tooltip, 축, 수치 포맷, 접근성 설명을 유지한다.

## 범위 제외

- 이전 기간 면 채움.
- 차트 데이터, 위치 정렬, tooltip 내용, 축 라벨 변경.
- ClickHouse 및 PostgreSQL 조회, 10분 캐시, 15분 rollup과 raw tail 변경.
- 다른 대시보드 차트 컴포넌트 변경이나 공통 차트 추상화 도입.
- 새로운 색상 토큰, 애니메이션 또는 외부 차트 라이브러리 추가.

## 승인된 시각 규칙

선택된 A안은 다음과 같다.

- 현재 기간
  - 선: `var(--color-chart-1)`.
  - 두께: 2px.
  - 면: 같은 색상의 세로 `linearGradient`.
  - 상단 불투명도: `0.32`.
  - 하단 불투명도: `0.04`.
  - 점: 표시하지 않음.
- 이전 기간
  - 선: `var(--color-muted-foreground)`.
  - 패턴: `4 4` 점선.
  - 면: 없음.
  - 점: 표시하지 않음.
- 두 시리즈 모두 애니메이션을 사용하지 않는다.

기존 `UsageAreaChart`는 단일 시계열에 상단 불투명도 `0.5`를 사용한다. 비교 그래프는 이전 기간 점선이 함께 보이므로 상단 불투명도를 `0.32`로 낮춰 교차 구간을 가리지 않는다. 하단 불투명도 `0.04`와 세로 방향은 동일하게 유지한다.

## 컴포넌트 구조

`apps/web/components/charts/insight-comparison-chart.tsx`만 변경한다.

현재 `LineChart`를 `ComposedChart`로 교체하고 다음 자식 요소를 배치한다.

1. SVG `defs` 안의 현재 기간 전용 `linearGradient`.
2. 기존 `CartesianGrid`, `XAxis`, `YAxis`, `Tooltip`.
3. `dataKey="current"`인 `Area`.
4. `dataKey="previous"`인 `Line`.

`Area`가 현재 기간의 실선과 면을 함께 그리므로 별도의 현재 `Line`을 중복 렌더링하지 않는다. 이전 기간만 `Line`으로 남긴다.

그라데이션 ID는 React `useId()`에서 파생하고 콜론을 제거해 만든다. 같은 페이지에 차트가 둘 이상 렌더링되더라도 SVG `url(#id)`가 다른 차트의 그라데이션을 참조하지 않게 한다.

## 데이터와 오류 처리

`InsightTrendPoint[]`를 `{ position, current, previous }`로 변환하는 기존 데이터 흐름을 그대로 유지한다. 비용·토큰 선택, compact 숫자 포맷, tooltip의 현재·이전 명칭도 변경하지 않는다.

빈 데이터와 저장소 오류 처리는 상위 인사이트 페이지의 기존 동작을 따른다. 이 변경은 렌더링 요소만 바꾸므로 별도 오류 상태나 폴백을 추가하지 않는다.

## 접근성

- `ComposedChart`의 `accessibilityLayer`를 유지한다.
- 기존 `aria-label`과 `aria-describedby`를 유지한다.
- 화면 밖 설명 문단과 번역 키를 유지한다.
- 그라데이션은 장식적 보조 표현이며, 현재·이전 구분은 실선과 점선에도 남아 색상만으로 정보를 전달하지 않는다.

## 테스트 기준

`apps/web/lib/ui-commonization.test.ts`의 차트 계약 테스트를 보강한다.

- `ComposedChart`를 사용한다.
- 현재 기간은 `Area`이며 `fill="url(#...)"`, `stroke="var(--color-chart-1)"`, `isAnimationActive={false}`를 가진다.
- 그라데이션은 상단 `0.32`, 하단 `0.04` 불투명도를 가진다.
- 이전 기간은 `Line`이며 회색 점선, 면 없음, `isAnimationActive={false}`를 유지한다.
- 숫자형 X축과 `dataMin`·`dataMax` 도메인을 유지한다.
- 접근 가능한 이름과 설명 연결을 유지한다.

## 실제 화면 검증

- `/insights?period=7`에서 토큰 그래프를 확인한다.
- `/insights?period=7&metric=cost`에서 비용 그래프를 확인한다.
- 라이트·다크 테마에서 현재 면과 이전 점선이 모두 구분되는지 확인한다.
- 390×844 모바일 viewport에서 그래프가 카드 폭을 넘지 않는지 확인한다.
- 변경 전후에 `/api/health`, `/api/ready`와 인사이트 데이터 응답이 동일하게 정상인지 확인한다.

## 완료 조건

- 현재 기간에만 승인된 옅은 그라데이션이 보인다.
- 이전 기간 점선과 tooltip·축·접근성 계약이 유지된다.
- 저장소, 캐시, rollup 및 다른 차트 파일은 변경되지 않는다.
- 관련 웹 테스트와 타입 검사, 실제 브라우저 검증이 통과한다.
