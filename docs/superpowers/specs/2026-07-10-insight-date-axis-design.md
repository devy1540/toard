# 인사이트 비교 그래프 날짜 축 설계

## 목표

인사이트 비교 그래프의 `1, 3, 5, 8` 경과 위치를 사용자가 바로 이해할 수 있는 실제 날짜로 바꾼다. 비교 정렬 방식과 집계 쿼리는 유지하고 표시 계층만 변경한다.

## 표시 규칙

- x축은 현재 기간의 시작 시각과 각 trend `position`을 사용해 계산한 현재 기간 날짜를 표시한다.
- 날짜는 뷰어의 locale과 timezone을 적용한 짧은 월/일 형식으로 표시한다.
- 이전 기간 데이터는 기존처럼 같은 경과 위치에 겹쳐 그린다.
- 툴팁 라벨에는 같은 위치의 현재 기간 날짜와 이전 기간 날짜를 함께 표시한다.
- 차트 공간이 좁으면 Recharts의 기존 눈금 생략 동작을 유지한다.

## 컴포넌트와 데이터 흐름

`insights/page.tsx`가 이미 계산한 `pair.current.from` 및 `pair.previous.from` 기간 시작 시각과 `pair.timezone`을 `InsightComparisonChart`에 전달한다. 차트는 trend의 0 기반 `position`에 해당 캘린더 일수를 더해 두 기간의 날짜를 계산한다.

데이터베이스와 저장소 계층의 `position` 계산은 변경하지 않는다. 따라서 PostgreSQL과 ClickHouse의 비교 집계 결과 및 서버 부하는 기존과 동일하다.

## 경계 조건

- 최근 7일처럼 첫날과 마지막 날이 일부만 포함되어 8개 캘린더 날짜에 걸치는 경우도 실제 날짜로 그대로 표시한다.
- DST가 포함된 기간은 24시간 밀리초를 단순 가산하지 않고, timezone 기준 캘린더 날짜에 `position`을 적용한다.
- 비어 있는 trend는 기존 빈 차트 동작을 유지한다.
- locale이나 timezone이 달라져도 해당 사용자 기준 날짜가 표시되어야 한다.

## 테스트

- 날짜 라벨 계산 함수를 분리해 현재·이전 기간의 동일 position이 올바른 날짜로 변환되는지 단위 테스트한다.
- DST 경계와 최근 7일의 8개 캘린더 날짜 케이스를 검증한다.
- 차트 계약 테스트로 숫자 position을 직접 노출하지 않고 날짜 tick formatter와 양쪽 기간 tooltip formatter를 사용하는지 확인한다.
- 기존 web/core 테스트와 web typecheck를 실행하고 실제 브라우저에서 데스크톱·모바일 x축 및 툴팁을 확인한다.
