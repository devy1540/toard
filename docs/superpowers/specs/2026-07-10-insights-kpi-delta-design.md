# 인사이트 KPI 증감 표시 공통화 설계

## 배경

인사이트 KPI 카드는 증감률을 `+17.4% (이전 기간 대비)` 같은 회색 텍스트로 직접 만들고 있다. 내 사용량·전체 현황·팀 현황은 `pctDelta`와 `DeltaBadge`를 사용해 화살표, 방향별 색상, 정수 반올림, 극단값 제한을 공통 적용한다. 이 때문에 같은 기간 비교 지표가 화면마다 다르게 보인다.

## 결정

인사이트 KPI의 증감 표시도 기존 `pctDelta`와 `DeltaBadge`를 재사용한다.

- 증가는 빨간색 상승 배지, 감소는 초록색 하락 배지로 표시한다.
- 증감률은 기존 공통 규칙대로 정수로 반올림한다.
- 이전 값이 0이거나 반올림 결과가 0%이면 배지를 표시하지 않는다.
- 극단적인 증감률은 기존 공통 규칙대로 ±999%에서 제한한다.
- 배지가 있으면 옆에 `이전 기간 대비` 문구를 유지한다.
- 배지가 없으면 기존 `이전 기간 데이터 없음` 문구를 유지해 비교 불가 상태를 설명한다.

시각만 배지로 바꾸고 소수점 한 자리 계산을 유지하는 안은 공통 규칙이 다시 갈라지므로 사용하지 않는다. 인사이트 전용 증감 컴포넌트를 새로 만드는 안도 중복이어서 제외한다.

## 구현 범위

- `apps/web/app/(dashboard)/insights/page.tsx`
  - `KpiCard`가 비교 문자열 대신 `StatDelta | null`과 비교 안내 문구를 받도록 변경한다.
  - 토큰·세션·비용 모두 `pctDelta`로 증감 데이터를 만든다.
  - `DeltaBadge`로 시각화하고 공통 카드와 같은 간격으로 안내 문구를 배치한다.
- `apps/web/lib/ui-commonization.test.ts`
  - 인사이트가 `pctDelta`와 `DeltaBadge`를 사용하고 자체 백분율 계산을 제거했다는 계약을 추가한다.

## 비범위

- 인사이트 후보 문장과 구성 비중의 `%p` 표시는 변경하지 않는다.
- KPI 값의 숫자·통화 포맷은 변경하지 않는다.
- 인사이트 기간 계산, 10분 캐시, PostgreSQL·ClickHouse 쿼리는 변경하지 않는다.
- 다른 대시보드 카드의 색상 규칙이나 증감 계산 규칙은 변경하지 않는다.

## 검증

1. 계약 테스트를 먼저 바꿔 기존 구현에서 실패하는지 확인한다.
2. 최소 구현 후 웹 테스트와 타입 검사를 통과시킨다.
3. 전체 워크스페이스 테스트와 `git diff --check`를 실행한다.
4. 로컬 인사이트 화면에서 증가·감소 배지가 기존 대시보드와 같은 색상·아이콘으로 표시되는지 확인한다.
