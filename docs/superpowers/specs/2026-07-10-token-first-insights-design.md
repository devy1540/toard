# 토큰 중심 인사이트 및 레이아웃 정렬 설계

## 배경

개인 인사이트 1차 구현은 비용을 기본 지표로 두고 비용, 세션, 토큰 순으로 정보를 배치했다. 실제 화면 비교 결과 인사이트의 페이지 제목은 24px이고 필터는 라벨이 위에 붙는 독립형 구조인 반면, `내 사용량`, `히스토리`, `설정`은 14px 제목과 한 줄 툴바를 사용한다. 인사이트만 시각적 문법이 달라 개인 메뉴 안에서 이동할 때 밀도와 정렬이 달라 보인다.

이번 변경은 인사이트를 토큰 중심으로 전환하면서, 향후 전체 대시보드 헤더 개편 전까지 현재 공통 화면 문법에 맞추는 국소 변경이다. 전체 화면의 공통 헤더 시스템을 새로 설계하거나 다른 페이지를 변경하지 않는다.

이 문서는 기존 [개인 인사이트 설계](./2026-07-10-personal-insights-design.md)의 화면 기본 지표, 정보 순서, 헤더 레이아웃을 대체한다. 기간 계산, 저장소 계약, ClickHouse 조회, 캐시, rollup 정책은 기존 설계를 유지한다.

## 목표

- `/insights`의 기본 관점을 비용이 아닌 토큰으로 변경한다.
- 토큰, 세션, 비용 순서로 KPI를 배치하고 비용은 보조 지표로 유지한다.
- 기본 인사이트 문장, 추이, 구성 변화를 토큰 기준으로 보여준다.
- 사용자가 비용을 명시적으로 선택하면 비용 기준 분석으로 전환한다.
- 인사이트 제목과 필터를 현재 대시보드의 작은 제목 및 한 줄 툴바 문법에 맞춘다.
- 기존 ClickHouse 쿼리 수, 10분 캐시, 15분 rollup과 raw tail 결합 정책을 그대로 유지한다.

## 범위 제외

- `내 사용량`, `히스토리`, `설정`, `전체 현황` 등 다른 페이지의 헤더 변경.
- 공통 `DashboardFilters`의 API 확장이나 전면 재설계.
- 비용 지표와 비용 선택 기능 제거.
- 새 집계 필드, 저장소 API, ClickHouse SQL, 캐시 키 또는 TTL 변경.
- 인사이트 규칙의 임계값과 최소 표본 수 변경.
- 직접 선택 기간, 90일 초과 기간, 팀 또는 조직 인사이트 추가.

## 승인된 접근

인사이트 전용 필터와 페이지 구조만 현재 공통 화면 문법에 맞춘다.

`DashboardFilters`를 직접 재사용하지 않는다. 인사이트는 `최근 7일`, `이번 주`, `이번 달`을 현재·이전의 같은 경과 길이로 비교하므로 일반 대시보드 기간 필터와 의미가 다르다. 공통 필터에 인사이트 전용 의미를 추가하면 기존 화면까지 영향을 받는다. 대신 기존 `InsightFilters`의 URL 갱신 동작과 기간 의미는 유지하면서 표현만 압축한다.

향후 전체 대시보드 헤더를 공통적으로 바꾸는 작업에서는 이 임시 정렬을 포함해 별도 설계한다.

## 화면 구조

### 상단 툴바

최상위 컨테이너의 `space-y-6`과 대시보드 레이아웃의 공통 패딩은 유지한다. 기존 24px 제목과 설명 블록을 제거하고 다음 요소를 한 줄의 유연한 툴바에 배치한다.

1. `인사이트` 14px 제목.
2. 기간: `최근 7일`, `이번 주`, `이번 달`.
3. provider 선택.
4. 지표: `토큰`, `비용` 순서.
5. 우측의 데이터 기준 시각과 최대 10분 지연 안내.

폭이 부족하면 요소는 자연스럽게 다음 줄로 감긴다. 화면에 보이는 `기간`, `도구`, `지표` 필드 라벨은 제거하되 각 컨트롤의 `aria-label`은 유지해 접근 가능한 이름을 보존한다.

현재 기간과 이전 기간의 정확한 범위는 툴바 아래의 보조 배경 영역에 두 칸으로 유지한다. 모바일에서는 한 열로 쌓인다.

### 토큰 중심 정보 계층

기본 URL에 유효한 `metric`이 없으면 `tokens`를 선택한다. `metric=tokens`와 `metric=cost`는 모두 유효하며 공유 URL에서 선택 상태를 보존한다.

기본 화면 순서는 다음과 같다.

1. 토큰 기준 핵심 변화 문장 최대 3개.
2. 토큰, 세션, 비용 KPI.
3. 현재·이전 기간의 토큰 추이.
4. 토큰 기준 모델 및 provider 구성 변화.

비용 KPI는 제거하지 않고 세 번째 카드에 둔다. 사용자가 비용을 선택하면 인사이트 후보, 추이, 모델 및 provider 구성은 모두 비용 기준으로 함께 전환한다. 이때도 KPI 순서는 토큰, 세션, 비용으로 유지해 페이지의 기본 정보 계층이 흔들리지 않게 한다.

규칙 엔진의 임계값, 최소 표본 수, 최대 후보 수는 바꾸지 않는다. 선택된 지표가 어떤 사용량 후보와 구성 변화 후보를 만들지 결정하는 기존 계약만 사용한다.

### 간격과 카드

- 주요 KPI와 상위 콘텐츠 그리드는 현재 대시보드에서 널리 쓰는 `gap-4`에 맞춘다.
- 요약 카드 내부 스타일, 차트 카드, 구성 변화 컴포넌트는 기존 UI 컴포넌트를 유지한다.
- 새로운 색상 토큰, 카드 변형, 차트 라이브러리는 추가하지 않는다.
- 토큰 중심성은 새로운 장식보다 선택 상태, 순서, 제목으로 표현한다.

## 데이터 흐름과 부하

요청 흐름은 기존과 동일하다.

1. URL에서 기간, provider, 지표를 해석한다.
2. 현재·이전 기간 쌍과 캐시 키를 만든다.
3. 캐시된 `UserInsightComparison`을 읽거나 저장소 집계를 수행한다.
4. 선택된 지표로 규칙 후보와 표시 데이터를 만든다.
5. 서버 컴포넌트가 토큰 중심 순서로 렌더링한다.

`metric`은 번역된 문장, 차트, 구성 비율을 선택하는 표시 관점이며 집계 DTO 캐시 키에 포함되지 않는다. 비용과 토큰은 동일한 DTO에 이미 들어 있으므로 기본 지표를 바꾸거나 사용자가 전환해도 추가 ClickHouse 조회가 생기지 않는다.

다음 정책은 변경하지 않는다.

- 인사이트 결과 캐시 TTL 10분.
- 최대 조회 범위 90일 제한.
- ClickHouse 캐시 미스당 데이터 쿼리 요청 최대 2개.
- 확정 구간의 15분 rollup과 부분 버킷 및 최신 raw tail 결합.
- dirty bucket 및 watermark 상태에 따른 raw 폴백.

## 오류와 빈 상태

로그인 필요, 현재 기간 데이터 없음, 이전 기간 데이터 없음, 구성 데이터 없음, 저장소 오류 처리는 기존 인사이트 동작을 유지한다. 토큰 기본값 때문에 비용 데이터만 있고 토큰이 없는 경우에도 현재 사용량 존재 여부는 비용, 세션, 토큰 전체를 기준으로 판단한다.

유효하지 않은 `metric` 값은 오류를 내지 않고 토큰 기본값으로 정규화한다. 사용자가 명시적으로 선택한 `metric=cost`는 비용 화면을 유지한다.

## 구현 경계

- `apps/web/app/(dashboard)/insights/page.tsx`
  - 기본 지표 해석.
  - 헤더 및 KPI 순서.
  - 주요 그리드 간격.
- `apps/web/components/dashboard/insight-filters.tsx`
  - 토큰/비용 선택 순서.
  - 한 줄 툴바용 압축 표현.
  - 기존 URL 파라미터 갱신과 접근 가능한 이름 유지.
- `apps/web/messages/ko.json`, `apps/web/messages/en.json`
  - 제거되는 설명 또는 시각 라벨 정리와 필요한 토큰 중심 문구 확인.
- 기존 인사이트 페이지 및 규칙 테스트
  - 기본값, 명시적 비용 선택, KPI 순서와 레이아웃 계약 보강.

저장소 패키지와 `@toard/core`는 변경 대상이 아니다.

## 테스트 기준

### 동작

- `metric`이 없거나 유효하지 않으면 토큰이 선택된다.
- `metric=cost`이면 비용 선택이 유지되고 비용 기준 요약, 차트, 구성이 렌더링된다.
- 지표 선택 순서는 토큰, 비용이다.
- KPI는 토큰, 세션, 비용 순서다.
- 기간과 provider를 변경해도 선택된 지표가 URL에 보존된다.

### 레이아웃과 접근성

- 인사이트 제목이 다른 개인 화면과 같은 14px 툴바 제목으로 렌더링된다.
- 기간, provider, 지표 컨트롤이 데스크톱에서 한 줄 툴바에 배치된다.
- 좁은 화면에서는 컨트롤과 freshness 정보가 겹치지 않고 줄바꿈된다.
- 시각 라벨을 제거한 뒤에도 각 컨트롤의 접근 가능한 이름이 유지된다.
- 현재·이전 기간 범위와 최대 10분 지연 안내가 계속 노출된다.
- 실제 브라우저에서 `내 사용량`, `히스토리`, `설정`과 제목 크기 및 상단 밀도를 비교한다.

### 회귀와 부하

- 기존 인사이트 규칙, 기간, 캐시, PostgreSQL 및 ClickHouse 테스트가 통과한다.
- 기본 지표 변경 전후에 인사이트 집계 쿼리 수가 늘지 않는다.
- 웹 타입 검사와 관련 테스트가 통과한다.
- `git diff --check`가 통과한다.

## 완료 조건

- `/insights` 기본 진입이 토큰 중심으로 보인다.
- 비용 선택 및 공유 URL이 계속 동작한다.
- 인사이트 상단이 현재 대시보드 화면의 제목과 필터 밀도에 맞는다.
- 저장소, 캐시, rollup 코드는 변경되지 않는다.
- 데스크톱과 모바일 실제 화면을 목업 및 다른 대시보드 화면과 비교해 확인한다.
