# 기존 저장 비용 표시 개선 설계

## 배경

가격 revision 도입 이전의 사용량은 당시 저장된 비용을 보존하며 `cost_status=legacy`로 집계한다. 현재 대시보드는 이 상태를 화면 상단의 파란 안내 배너와 비용 값의 `기존 저장 비용` 접미사로 동시에 표시한다.

`legacy` 비용은 가격 근거가 없는 미확정 비용이 아니다. 이미 저장된 금액이 합계에 포함된 정상적인 과거 데이터다. 그런데 현재 표현은 상단 공간을 크게 차지하고 비용 KPI 자체도 길게 만들어, 사용자가 현재 비용의 오류나 조치가 필요한 상태로 오해할 수 있다.

## 목표

- 비용 합계를 가장 먼저 읽을 수 있게 한다.
- `legacy` 비용의 출처는 숨기지 않되 보조 정보로 낮춘다.
- 실제 조치가 필요한 `unpriced`·`partial` 상태와 정상적인 과거 데이터인 `legacy` 상태의 시각적 위계를 분리한다.
- 개인, 인사이트, 조직, 팀 화면에서 같은 가격 상태를 같은 방식으로 표현한다.
- 사용자 화면에서 `legacy`, `revision` 같은 내부 용어를 제거한다.

## 범위

다음 화면에 공통 적용한다.

- 개인 대시보드 개요 뷰
- 개인 대시보드 클래식 뷰
- 인사이트
- 조직 개요
- 조직 팀 목록
- 팀 상세

가격 집계 방식, 저장된 비용 값, 가격 동기화, rollup 생성과 조회 경로는 변경하지 않는다.

## 확정 디자인

### 1. 가격 상태별 위계

| 상태 | 비용 값 | 화면 상단 알림 | 비용 KPI 보조 문구 |
| --- | --- | --- | --- |
| `complete` | 확정 비용 숫자 | 없음 | 없음 |
| `legacy` | 저장된 비용 숫자 | 없음 | `{count}건은 이전 가격 기준` |
| `partial` | 비용 숫자와 `부분 합계` | 경고 유지 | 기존 경고 설명 유지 |
| `unpriced` | `가격 미확정` | 경고 유지 | 기존 경고 설명 유지 |

`legacy`는 오류나 미완료 상태가 아니므로 상단 정보 배너를 표시하지 않는다. `partial`과 `unpriced`는 합계가 불완전하고 관리자의 가격 동기화가 필요할 수 있으므로 기존 경고와 관리자 동선을 유지한다.

### 2. 비용 값

대표 비용 KPI는 `$372.44`처럼 숫자만 주 값으로 표시한다. `legacy` 상태일 때 현재의 `$372.44 · 기존 저장 비용` 접미사는 제거한다.

모델, 기기, 팀 등 반복 목록의 각 비용 값에도 `기존 저장 비용`을 덧붙이지 않는다. 같은 상태 설명을 모든 행에 반복하지 않고, 해당 화면의 대표 비용 KPI 한 곳에서 집계 범위 전체를 설명한다.

### 3. 보조 문구

`legacyEvents > 0`이고 `unpricedEvents === 0`이면 대표 비용 KPI 바로 아래에 다음 문구를 표시한다.

> 3,397건은 이전 가격 기준

건수는 현재 조회 범위와 필터가 반환한 `costCoverage.legacyEvents`를 사용한다. 색상과 크기는 기존 KPI의 힌트 텍스트 수준으로 낮추고, 경고 아이콘이나 경고색은 사용하지 않는다.

인사이트 비교 화면의 비용 KPI는 현재 기간의 비용을 보여주므로 보조 문구도 `comparison.current.costCoverage.legacyEvents`만 사용한다. 이전 기간의 건수를 더하지 않는다. 현재·이전 기간 중 하나라도 가격 미확정 사용량이 있으면 기존 비교 경고 규칙을 우선한다.

`partial` 또는 `unpriced` 상태에서는 이 보조 문구를 별도로 표시하지 않는다. 상단 경고가 미확정 건수와 합계의 불완전성을 이미 설명하므로 중복을 피한다.

### 4. 문구

한국어 사용자 문구는 `이전 가격 기준`으로 통일한다. 영어는 `using earlier stored prices`처럼 같은 의미의 사용자 언어를 사용한다. 사용자 화면에는 `legacy`, `revision`, `provenance`를 노출하지 않는다.

## 컴포넌트 책임

- `costCoverageState`는 현재의 네 상태 판정을 유지한다.
- 비용 문자열 포매터는 `legacy`일 때 비용 숫자를 그대로 반환한다. `partial`과 `unpriced` 처리는 유지한다.
- 가격 알림 컴포넌트는 `complete`와 `legacy`에서 렌더링하지 않고, `partial`과 `unpriced`에서만 렌더링한다.
- 대표 비용 KPI는 공통 보조 문구 생성 규칙을 사용한다. 화면별로 상태 조건과 문구를 다시 구현하지 않는다.
- 반복 목록과 차트 툴팁은 공통 비용 포매터를 사용해 `legacy` 접미사를 제거한다.

## 데이터 흐름

1. 기존 조회가 `totalCostUsd`와 `costCoverage`를 함께 반환한다.
2. `costCoverageState`가 표시 상태를 판정한다.
3. 비용 포매터가 상태에 따라 주 값을 결정한다.
4. `legacy` 상태이면 대표 비용 KPI가 `legacyEvents` 건수를 보조 문구로 표시한다.
5. `partial` 또는 `unpriced`이면 가격 알림이 기존 경고와 관리 동선을 표시한다.

추가 API 호출이나 가격 테이블 조회는 만들지 않는다.

## 예외와 오류 처리

- `legacyEvents`가 0이면 보조 문구를 렌더링하지 않는다.
- `legacyEvents`와 `unpricedEvents`가 함께 있으면 `partial`을 우선하고 기존 경고를 표시한다.
- 비용 합계가 0이어도 `legacyEvents > 0`이면 `$0.00`과 이전 가격 기준 건수를 그대로 표시한다. 저장된 비용이 실제로 0일 수 있으므로 상태를 임의로 바꾸지 않는다.
- 번역 키가 누락되지 않도록 한국어와 영어 메시지를 함께 변경한다.

## 테스트와 검증

### 단위 테스트

- `legacy` 비용 포매팅이 접미사 없이 비용 숫자만 반환한다.
- `partial`과 `unpriced` 포매팅은 기존 동작을 유지한다.
- `legacy` 상태에서는 상단 알림이 렌더링되지 않는다.
- `partial`과 `unpriced` 상태에서는 기존 경고가 렌더링된다.
- 이전 가격 기준 보조 문구는 순수 `legacy` 상태에서만 생성된다.
- 한국어와 영어 번역 키가 모두 존재한다.

### 화면 검증

- 개인 개요·클래식, 인사이트, 조직 개요, 팀 목록·상세에서 상단 `legacy` 배너가 사라진다.
- 각 화면의 대표 비용 KPI는 비용 숫자를 주 값으로 표시하고 그 아래에 이전 가격 기준 건수를 표시한다.
- 반복 비용 행에는 `기존 저장 비용` 접미사가 나타나지 않는다.
- 좁은 폭에서 비용 숫자와 보조 문구가 다른 KPI를 침범하거나 잘리지 않는다.
- `partial`과 `unpriced` 시드 데이터에서는 기존 경고와 관리 링크가 유지된다.

### 회귀 검증

- 관련 웹 단위 테스트
- 웹 타입 검사
- 웹 프로덕션 빌드
- `git diff --check`

## 완료 기준

- 정상적인 과거 비용 데이터가 경고 배너로 보이지 않는다.
- 비용 KPI의 주 값은 접미사 없는 숫자로 읽힌다.
- 사용자는 같은 KPI 아래에서 이전 가격 기준 사용량 건수를 확인할 수 있다.
- 가격 미확정 데이터의 경고와 관리자 조치 경로는 유지된다.
- 대상 화면 전체가 같은 상태 판정, 포매팅, 문구 규칙을 공유한다.
