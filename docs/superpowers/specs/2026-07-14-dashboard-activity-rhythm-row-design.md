# 내 사용량 활동·리듬 한 줄 배치 설계

## 목표

E2EE 전환 뒤 의미가 없고 잘못된 `server_v1` 라벨을 표시하는 `최근 세션` 카드를 내 사용량 개요에서 제거한다. 남은 `AI 도구 활동`과 `시간대 리듬`을 데스크톱 한 줄에 균형 있게 배치한다.

## 선택한 배치

- 데스크톱 `xl` 이상: `AI 도구 활동 55% | 시간대 리듬 45%`
- `xl` 미만: `AI 도구 활동` 다음 `시간대 리듬` 순서로 세로 배치
- 두 카드는 같은 grid 행에서 각각 독립적인 카드 경계를 유지한다.
- AI 도구 활동은 지표 3개와 상위 활동 태그 때문에 더 넓은 열을 사용한다.

## 데이터 흐름

`OverviewView`는 더 이상 `getMyHistorySessions`를 호출하거나 최근 세션의 usage 요약을 조회하지 않는다. 내 사용량 개요는 집계 사용량, 시간대별 사용량, 토큰 메타데이터와 AI 도구 활동만 조회한다. 본문 히스토리와 세션 탐색은 `/history`에만 둔다.

방어적으로 레거시 히스토리 조회 SQL은 `encryption_scheme = 'server_v1'` 조건을 사용해 E2EE 행을 레거시 경로에 섞지 않는다. 이 조건은 E2EE 비활성 사용자의 기존 기록 호환에는 영향을 주지 않는다.

## 컴포넌트 변경

- `ToolActivityCard`는 선택적인 `className`을 받아 grid 열 안에서 높이와 너비를 맞출 수 있게 한다.
- `OverviewView`는 `ToolActivityCard`와 시간대 리듬 section을 `xl:grid-cols-[minmax(0,1.2fr)_minmax(0,1fr)]`에 함께 배치한다.
- 최근 세션 전용 상수, imports, 조회, 비용 조인, 렌더링을 삭제한다.
- 더 이상 사용되지 않는 최근 세션 번역 키를 한국어·영어에서 함께 삭제한다.

## 검증

- 정적 UI 회귀 테스트가 최근 세션 조회·라벨·카드가 사라졌는지 확인한다.
- 같은 테스트가 활동/리듬 grid와 1.2fr:1fr 비율, 모바일 렌더 순서를 확인한다.
- 레거시 히스토리 테스트가 `server_v1` 필터 존재를 확인한다.
- 전체 웹 테스트, 타입 검사, 프로덕션 빌드를 실행한다.
