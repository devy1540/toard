# E2EE 히스토리 목록 복원 설계

## 배경

현재 `/history`는 E2EE 전용 목록을 기존 히스토리 목록 위에 추가로 렌더링한다. 전용 목록은 미리보기와 턴 수만 한 줄로 노출하고 기간·도구 필터, 날짜 구분, 모델·토큰·기기·시간 메타데이터, 페이지네이션을 제공하지 않는다. 그 결과 같은 기록이 서로 다른 디자인으로 중복 노출되고 긴 프롬프트가 목록 너비를 밀어낸다.

또한 `/history` 전용 CSP가 `require-trusted-types-for 'script'`를 즉시 강제한다. 새 문서로 로드되는 새로고침에서는 React와 Cloudflare가 삽입한 이메일 보호 스크립트의 `innerHTML` 사용이 차단되어 클라이언트 초기화가 중단된다. 다른 페이지에서의 클라이언트 이동은 기존 문서의 정책을 유지하므로 같은 오류가 드러나지 않는다.

## 목표

- E2EE가 활성화된 계정에는 히스토리 목록을 하나만 표시한다.
- 기존 히스토리의 정보 구조와 시각 디자인을 E2EE 목록에도 동일하게 적용한다.
- 전체·오늘·이번 주·이번 달·3개월·12개월·직접 선택과 도구 필터를 지원한다.
- 날짜별 구분, 도구 아이콘·배지, 모델 배지, 2줄 미리보기, 턴·토큰·기기·비용·시간을 표시한다.
- 한 페이지에 20건을 표시하고 전체 건수와 이전·다음 페이지를 제공한다.
- 프롬프트·응답 평문은 브라우저에서만 복호화한다.
- `/history` 직접 진입과 새로고침이 클라이언트 오류 없이 동작한다.
- 좁은 화면에서는 메타데이터가 자연스럽게 줄바꿈되고 긴 미리보기나 URL이 가로 스크롤을 만들지 않는다.

## 비목표

- E2EE 키 구조, 복구키, 승인 기기 흐름을 변경하지 않는다.
- 서버가 프롬프트·응답 평문을 복호화하지 않는다.
- 기존 히스토리 상세 화면의 전체 재설계는 이번 범위에 포함하지 않는다. E2EE 상세의 현재 복호화·표시 동작은 유지한다.

## 화면 구조

E2EE 활성 계정의 목록 화면은 다음 순서로 구성한다.

1. `내 히스토리` 제목과 `프리뷰` 상태 배지
2. 기간·도구 필터
3. 우측의 작은 E2EE 잠금 해제 상태와 `지금 잠그기` 동작
4. 날짜별로 묶인 세션 목록
5. 전체 건수와 페이지네이션

E2EE 상태는 별도 대형 섹션 제목으로 반복하지 않는다. 잠금 해제 상태와 마이그레이션 상태는 툴바의 보조 정보로 축약한다. 잠긴 상태, 승인 대기, 복구 흐름은 기존 `LockedHistory`를 그대로 사용한다.

각 목록 행은 기존 히스토리와 동일한 계층을 갖는다.

- 왼쪽: 도구 아이콘
- 첫 줄: 도구 배지, 모델 배지 최대 2개, 초과 모델 수
- 둘째 줄: 브라우저에서 복호화한 미리보기, 최대 2줄
- 셋째 줄: 턴 수, 토큰 수, 기기
- 오른쪽: 비용과 마지막 활동 시간

## 데이터 흐름

### 서버 페이지

서버 페이지는 로그인 사용자의 E2EE 상태를 확인한다.

- `active`: E2EE 클라이언트 화면만 렌더링한다.
- `off` 또는 계정 없음: 기존 서버 복호화 히스토리를 렌더링한다.
- `pending`: E2EE 설정/잠금 화면만 렌더링해 두 데이터 소스를 동시에 노출하지 않는다.

이 분기로 현재의 E2EE 목록과 기존 목록 중복을 제거한다.

### E2EE 세션 API

`/api/content/history/sessions`는 URL의 기간·직접 선택·도구·페이지 값을 받아 서버에서 암호문 행의 메타데이터만 필터링한다. 응답에는 다음을 포함한다.

- 세션 키, 실제 세션 여부, 도구, 턴 수, 최초·최종 시각
- 브라우저가 복호화할 대표 암호문 1건
- 이미 서버에 존재하는 사용량 메타데이터: 모델, 토큰, 기기, 비용·가격 커버리지
- 필터에 해당하는 전체 세션 수

평문 미리보기는 응답하지 않는다. 브라우저는 현재 페이지의 최대 20개 대표 암호문만 복호화한다.

### 클라이언트

필터가 바뀌면 URL을 갱신하고, E2EE 클라이언트가 해당 검색 파라미터로 세션 API를 다시 호출한다. 응답의 대표 암호문을 로컬 UCK로 복호화하고 `toHistoryPreview`로 표시 문자열을 만든다. 개별 복호화 실패는 해당 행만 `내용을 표시할 수 없음`으로 표시하며 전체 목록을 중단하지 않는다.

## CSP 수정

nonce 기반 `script-src`와 나머지 CSP는 유지한다. 현재 애플리케이션과 주입 스크립트가 완전하게 Trusted Types 정책을 사용하지 않으므로 `require-trusted-types-for 'script'` 강제 지시자는 제거한다. Cloudflare가 인증 화면 HTML을 변형해 이메일 보호 스크립트를 삽입하지 않도록 `/history` 응답의 `Cache-Control`에 `no-transform`을 추가한다.

Trusted Types는 향후 React·Next.js·외부 스크립트 전체가 명시적 정책을 사용하는 별도 작업에서 Report-Only 검증 후 다시 강제한다. 단순한 전역 기본 정책으로 모든 문자열을 허용해 보안 효과를 무력화하는 방식은 사용하지 않는다.

## 오류 처리

- API 실패: 기존 E2EE 오류 카드와 다시 잠그기 동작을 유지한다.
- 대표 암호문 복호화 실패: 해당 행만 대체 문구를 표시한다.
- 사용량 매칭 실패: 토큰·비용·기기 영역에 기존 `정보 없음` 표현을 사용한다.
- 필터 결과 0건: 수집 기록 없음과 조건 불일치 빈 상태를 구분한다.
- 페이지 범위를 벗어나면 빈 목록 대신 가능한 마지막 페이지로 이동하거나 이전 페이지 동작을 제공한다.

## 검증 기준

- `/history`를 주소창에서 직접 열거나 새로고침해도 전체 화면 클라이언트 오류가 발생하지 않는다.
- E2EE 활성 상태에서 목록 카드가 하나만 렌더링된다.
- 두 번째 참조 화면과 같은 필터·날짜 그룹·행 정보 계층이 보인다.
- 긴 URL·Markdown·XML 형태의 미리보기가 2줄에서 잘리고 가로 스크롤을 만들지 않는다.
- 기간·도구 필터와 페이지 이동 시 요청 조건과 표시 결과가 일치한다.
- 평문은 네트워크 응답과 서버 렌더 HTML에 포함되지 않는다.
- E2EE 상태·API·표시 순수 로직 테스트, 타입 검사, 프로덕션 빌드가 통과한다.
- 데스크톱과 좁은 폭에서 실제 화면을 확인한다.
