# Recovery Kit 번호형 레이아웃 설계

## 목적

24개 Recovery Kit 단어의 순서를 한눈에 확인하고, 저장 확인 단계에서 요청된 번호와 단어를 실수 없이 대응할 수 있게 한다.

## 화면 구성

- 각 단어를 `01 단어`부터 `24 단어`까지 번호가 포함된 독립 항목으로 표시한다.
- 데스크톱은 4열 6행, 모바일은 2열 12행으로 재배치한다.
- 번호는 고정 폭으로 표시하고 단어와 시각적으로 구분한다.
- 기존 경고문, 저장 버튼, 확인 폼의 순서와 주요 동작은 유지한다.
- 확인 입력 라벨은 `07번 단어`처럼 단어 목록과 동일한 두 자리 번호 체계를 사용한다.

## 저장 파일

- 다운로드 파일은 한 줄에 하나씩 `01. 단어` 형식으로 저장한다.
- 파일에는 제목과 24개 번호형 단어 외의 계정·기기·서버 정보가 포함되지 않는다.
- 실제 Recovery Kit 단어는 서버 요청, 애플리케이션 로그, 테스트 fixture에 기록하지 않는다.

## 구현 범위

- `shim/rust/src/e2ee_setup_page.html`에서 번호형 반응형 목록과 스타일을 제공한다.
- `shim/rust/src/e2ee_setup.rs`에서 안전하게 escape한 번호형 단어 항목과 확인 라벨을 렌더링한다.
- 저장 스크립트는 DOM의 24개 단어를 순서대로 읽어 번호가 포함된 텍스트 파일을 만든다.
- 암호화, Recovery Kit 생성, 확인 위치 선택, 활성화 API 계약은 변경하지 않는다.

## 검증

- 렌더링 결과에 24개 번호형 항목과 `01`, `24` 경계 번호가 존재한다.
- 요청 위치는 화면과 동일한 두 자리 번호로 표시된다.
- 저장 스크립트가 `01. 단어` 형식을 생성한다.
- HTML에는 `Cache-Control: no-store`와 기존 CSP가 유지된다.
- Rust fmt, Clippy, 단위 테스트, release build를 통과한다.
