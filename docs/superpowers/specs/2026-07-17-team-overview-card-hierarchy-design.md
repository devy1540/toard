# 팀별 현황 카드 위계 통일 설계

## 배경

`팀별 현황`은 상단 요약 지표를 배경 없는 `SummaryTile` 네 개로 표시하고, 이를 감싸는 콘텐츠 래퍼도 `contents`를 사용한다. 반면 `팀 현황`은 핵심 요약을 하나의 대표 카드로 묶고 아래 분석 영역을 독립된 카드로 나눈다.

이 차이 때문에 `팀별 현황`의 상단 지표가 필터 영역에 붙어 떠 있는 것처럼 보이고, `상위 팀`, `비용 분포`, `팀 상세 순위` 카드 사이의 시각적 위계도 `팀 현황`보다 약하게 느껴진다.

## 목표

`팀별 현황`의 데이터와 기능은 유지하면서 `팀 현황`과 같은 카드 문법과 수직 리듬을 적용한다.

- 상단 요약이 하나의 명확한 대표 카드로 인식되어야 한다.
- `상위 팀`, `비용 분포`, `팀 상세 순위`가 서로 다른 분석 섹션으로 즉시 구분되어야 한다.
- 두 메뉴를 이동했을 때 같은 제품군의 대시보드라는 일관성이 느껴져야 한다.
- 팀이 한 개뿐인 현재 데이터와 여러 팀이 있는 실제 운영 데이터 모두 자연스럽게 표시되어야 한다.

## 선택한 접근

`팀 현황`의 `TeamHero` 구조를 기준으로 `팀별 현황` 전용 대표 요약 카드를 만든다. 기존 순위 데이터와 하위 분석 컴포넌트는 유지하고 레이아웃 위계만 조정한다.

### 대표 요약 카드

- `순위 비용`을 왼쪽의 주 지표로 크게 표시한다.
- `순위 대상`, `순위 세션`, `1위 점유율`은 오른쪽의 세 열 보조 지표로 배치한다.
- 전체를 `bg-card`, 테두리, `rounded-xl`이 적용된 하나의 표면으로 묶는다.
- 기존 네 개 지표의 값과 설명 문구는 변경하지 않는다.
- 별도의 증감률이나 신규 데이터는 추가하지 않는다.

이 방식은 네 지표를 각각 독립 카드로 만드는 방식보다 화면이 잘게 쪼개지지 않고, `팀 현황` 상단 대표 카드와도 직접 대응된다.

### 분석 섹션

- 데스크톱에서는 `상위 팀`과 `비용 분포`를 같은 행의 두 카드로 배치한다.
- 두 카드는 동일한 카드 헤더 패딩과 제목·설명 간격을 사용한다.
- `상위 팀` 카드는 순위, 팀명, 세션, 토큰, 비용, 점유율을 한 그룹 안에서 읽게 한다.
- `비용 분포`는 기존 차트와 데이터를 유지한다.
- `팀 상세 순위`는 다음 행의 전체 너비 카드로 유지한다.
- 카드 사이 간격은 `gap-4`, 대표 카드와 다음 분석 행 사이의 수직 간격은 페이지의 `space-y-6` 규칙을 사용한다.

### 내부 행 표현

- `PodiumCard`와 `RankingListRow`의 기존 내부 테두리는 유지한다. 여러 팀일 때 반복 항목을 구분하는 역할이 있기 때문이다.
- 1위 강조 색상, 진행 막대, 비용 커버리지 표시는 변경하지 않는다.
- 데이터가 한 팀뿐이어도 상위 팀 카드가 과도하게 넓고 비어 보이지 않도록 비용 분포 카드와 같은 행을 구성한다.

## 컴포넌트 경계

### `TeamRankingHero`

`apps/web/app/(dashboard)/org/teams/page.tsx` 내부의 전용 표현 컴포넌트로 둔다.

- 합산 비용, 순위 대상 수, 합산 세션, 1위 점유율을 받는다.
- 데이터 조회나 합산을 수행하지 않는다.
- 대표 카드의 레이아웃과 표시만 책임진다.
- `팀 현황`의 `TeamHero`와 시각 문법은 맞추되 데이터 의미가 달라 직접 공통 컴포넌트로 추출하지 않는다.

### `AllTeamsOverview`

- 기존 `SummaryTile` 네 개의 그리드를 `TeamRankingHero`로 교체한다.
- 기존 `PricingNotice`, 리더보드 조회, 비용 커버리지 계산은 그대로 유지한다.
- `contents` 래퍼를 제거하고 명시적인 `space-y-6` 콘텐츠 스택을 사용한다.
- 하위 카드의 데이터와 렌더링 순서는 유지한다.

## 반응형 규칙

- 넓은 화면에서 대표 카드의 주 지표와 보조 지표를 좌우로 배치한다.
- 중간 화면에서는 보조 지표 세 열을 유지하되 대표 카드 안에서 전체 너비를 사용한다.
- 좁은 화면에서는 주 지표 아래에 보조 지표가 쌓이며 가로 스크롤이 생기지 않아야 한다.
- `상위 팀`과 `비용 분포`는 기존 브레이크포인트에서 한 열로 쌓인다.
- 반복 순위 행의 기존 모바일 그리드 전환은 유지한다.

## 변경하지 않는 범위

- 리더보드 조회, 비용·토큰·세션 합산 및 순위 계산
- 기간, 프로바이더, 자동 새로고침 필터
- 번역 문구와 메뉴 명칭
- 차트 데이터와 차트 축 계산
- `팀 현황`, `전체 현황` 및 공통 `Card` 컴포넌트
- 전역 색상 토큰과 사이드바 디자인

## 테스트와 검증

1. `팀별 현황`이 배경 없는 상단 네 칸 그리드 대신 대표 카드 컴포넌트를 사용한다는 계약 테스트를 추가한다.
2. 대표 카드에 네 기존 지표가 모두 전달되는지 검증한다.
3. `AllTeamsOverview`가 `space-y-6` 스택을 사용하고 분석 카드 사이의 `gap-4`를 유지하는지 검증한다.
4. 웹 타입 검사와 관련 테스트를 실행한다.
5. `git diff --check`로 공백 오류를 확인한다.
6. 실제 브라우저에서 `팀별 현황`과 `팀 현황`의 상단 카드 배경, 테두리, 라운드, 수직 간격을 나란히 비교한다.
7. 데스크톱과 390px 화면에서 카드 겹침, 잘림, 가로 스크롤이 없는지 확인한다.
