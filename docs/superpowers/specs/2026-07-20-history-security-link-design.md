# 내 히스토리 보안 안내 링크 설계

## 목표

`내 히스토리` 목록과 상세 화면에서 길게 반복되는 보안 설명을 제거한다. 대신 짧은 `보안 안내` 링크를 제공하고, 상세한 암호화·운영자 신뢰 경계는 기존 `설정 > 계정 > 히스토리 보안` 카드에서 확인하게 한다.

## 현재 문제

- 목록 상단에 본인 전용 안내와 DB·백업·KMS 설명이 연속으로 노출되어 페이지 제목보다 시각적 비중이 크다.
- 상세 화면에도 같은 설명이 반복된다.
- 동일한 신뢰 경계가 설정의 `히스토리 보안` 카드에도 이미 존재한다.

## 확정안

시각화의 C안을 적용한다.

1. 목록 툴바의 긴 설명을 방패 아이콘과 `보안 안내`로 구성된 작은 링크로 교체한다.
2. 상세 화면 제목 행에도 같은 링크를 배치한다.
3. 링크는 `/settings?tab=account#history-security`로 이동한다.
4. 설정의 기존 `히스토리 보안` 카드에 `history-security` 앵커를 부여한다.
5. DB·백업·KMS 신뢰 경계 문구는 설정 카드에 그대로 유지한다.

## 구성 요소

### `HistorySecurityLink`

목록과 상세 화면이 같은 목적지·아이콘·버튼 스타일을 사용하도록 작은 공유 컴포넌트로 둔다. 컴포넌트는 번역된 라벨을 받아 Next.js `Link`로 설정의 앵커를 연다.

### 히스토리 목록

`DashboardFilters`의 `trailing`에는 기존 여러 문장 대신 `HistorySecurityLink`만 전달한다. 콘텐츠 암호화 유형에 따른 문구 분기는 제거한다.

### 히스토리 상세

`SessionDetail` 내부의 반복 보안 문구를 제거한다. 페이지 제목과 프리뷰 배지가 있는 상단 행의 우측에 `HistorySecurityLink`를 배치한다. 따라서 상세 데이터 조회 결과에서 안내 렌더링에만 사용하던 `hasManagedContent`와 `hasLegacyContent` 구조 분해도 제거한다.

### 설정 화면

`HistorySecurityPanelView`의 최상위 카드에 `id="history-security"`를 지정한다. 기존 보호 상태, 키 버전, 운영자 신뢰 경계 문구와 legacy 상태 표시는 변경하지 않는다.

## 번역

대시보드의 한국어·영어 `history` 번역에 짧은 링크 라벨을 둔다.

- 한국어: `보안 안내`
- 영어: `Security info`

히스토리 화면에서 더 이상 사용하지 않는 `privacyNote`, `managedPrivacyNote`, `legacyPrivacyNote`는 양 언어에서 제거한다. 상세한 신뢰 경계는 `settings.historySecurity.privacyBoundary`가 계속 담당한다.

## 데이터와 보안

데이터 조회, 암호화, 권한 검사와 저장 형식은 변경하지 않는다. 이번 변경은 설명의 위치와 탐색 경로만 바꾸며, 사용자별 히스토리 접근 제어에는 영향을 주지 않는다.

## 검증

- 목록과 상세 화면이 모두 공통 `HistorySecurityLink`를 렌더링하는지 확인한다.
- 링크 목적지가 `/settings?tab=account#history-security`인지 확인한다.
- 설정 카드에 `history-security` 앵커가 존재하는지 확인한다.
- 한국어·영어 번역 키가 대칭인지 확인한다.
- 긴 신뢰 경계 문구가 히스토리 화면에서는 사라지고 설정 화면에는 남아 있는지 확인한다.
- 관련 단위 테스트, 타입 검사와 린트를 실행한다.

## 제외 범위

- 암호화 방식 또는 KMS 설정 변경
- 히스토리 권한 모델 변경
- 설정 카드의 상태·legacy 정보 재설계
- 별도 팝오버나 모달 추가
