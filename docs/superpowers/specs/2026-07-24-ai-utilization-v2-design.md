# AI 활용 지수 v2 설계

## 배경

`utilization-v1`은 컨텍스트 연속성, 실행 안정성, 복구 부담의 유효 점수를 가변 평균했다. 이 구조에서는 같은 도구 실패가 실행 안정성과 복구 부담에 중복 반영되고, 계산 가능한 축의 수에 따라 같은 종합점수의 구성이 달라진다. 화면도 원시 비율과 표본 수를 충분히 설명하지 않아 사용자가 점수의 근거를 확인하기 어렵다.

## 결정

- 방법론 버전을 `utilization-v2`로 올린다.
- 종합지수는 `컨텍스트 재사용률`과 `도구 실행 성공률`의 고정된 두 축으로만 계산한다.
- 두 축 중 하나라도 계산할 수 없으면 종합지수를 제공하지 않는다.
- 실패 후 30분 이내 같은 세션·도구의 결과 확인 재시도를 `복구 시도`로 정의한다.
- 복구 성공률과 연속 실패 건수는 진단값으로만 표시하고 종합지수에 중복 반영하지 않는다.
- 현재값, 직전 28일 중앙값, 차이, 현재·기준 표본 수, 결과 확인 범위를 함께 표시한다.
- 개인 화면은 최근 12주를 동일한 7일 현재 기간·28일 기준 기간으로 재계산해 추세를 표시한다.

## provider capability

- 캐시 신호: `claude_code`, `codex`, `cursor`, `gemini`, `qwen`
- 도구 결과: `claude_code`, `codex`, `cursor`
- 지원되지 않는 provider의 도구 이벤트는 성공·실패 분자와 분모에서 제외한다.
- `unknown`은 성공률 분모에서 제외하되 결과 확인 범위에는 포함한다.

## 화면 계약

개인 카드의 정보 순서는 다음과 같다.

1. 최근 7일의 자기 기준 상태와 상대 지수
2. 컨텍스트 재사용률과 도구 실행 성공률의 원시 비율·기준값·차이·표본
3. 실패 후 복구율 진단
4. 활성일·세션·결과 확인 호출·결과 확인 범위
5. 동일 방법론의 최근 12주 추세
6. 점수에 반영하지 않는 도구 사용 세션 비율과 도구 종류
7. 현재·기준 기간, 방법론 버전, 계산 시각

데이터가 부족한 상태에서는 0을 만들지 않고 이유를 표시한다. 45~55는 `내 평소 범위`로 설명하며, 높고 낮음을 업무 성과나 품질로 표현하지 않는다.

## 데이터 경계

프롬프트·응답·코드·도구 인자와 결과 본문은 읽지 않는다. 콘텐츠 수집 설정과 `prompt_records`의 에이전트 메타데이터도 점수와 신뢰도에 사용하지 않는다. 토큰·세션·도구 종류가 많다는 이유로 점수를 높이지 않는다.

## 검증 기준

- 두 점수 축만 종합지수에 고정 반영된다.
- 복구 진단값을 변경해도 두 점수 축이 같으면 종합지수는 변하지 않는다.
- Cursor 도구 결과가 capability와 집계 범위에 함께 포함된다.
- 도구 결과를 지원하지 않는 provider는 도구 축의 분자·분모에 포함되지 않는다.
- 과거 추세의 각 점은 해당 시점의 완료 7일과 직전 28일을 사용한다.
- 한영 화면이 같은 메시지 구조와 데이터 부족 상태를 제공한다.
- 개인 화면과 조직 화면은 개인 순위나 콘텐츠 데이터를 노출하지 않는다.
