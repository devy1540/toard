# AI 도구 메타데이터 수집

toard shim은 사용량 수집과 함께 MCP·스킬 활동 및 기기별 플러그인·스킬·MCP 설치 현황을 수집한다. 이 기능은 기본 활성화되며, 도구가 무엇이었는지와 언제 동작했는지를 보여주기 위한 메타데이터만 전송한다.

## 전송하는 정보

활동 이벤트(`/api/v1/tool-events`)는 다음 필드만 전송한다.

- 중복 제거용 해시, AI 프로바이더, 세션 식별자, 기기명, 발생 시각
- 활동 종류(`mcp` 또는 `skill`), 항목 키와 표시 이름
- 로그에서 출처를 명확히 확인할 수 있을 때의 플러그인 키
- 결과(`success`, `failure`, `unknown`)
- 감지 방식(`explicit`, `derived_load`)

기기 인벤토리(`/api/v1/tool-inventory`)는 다음 필드만 전송한다.

- 기기명, 관찰 시각, 변경 여부를 확인하는 fingerprint
- 설치 항목의 종류, 이름, 소스 프로바이더, 플러그인 키, 버전, 활성 상태

## 전송하지 않는 정보

AI 도구 메타데이터 경로는 아래 내용을 직렬화하거나 서버로 보내지 않는다.

- MCP 또는 다른 도구의 인자와 출력
- 셸 명령과 환경변수
- 로컬 파일의 절대 경로
- 원본 로그 또는 원본 payload
- 프롬프트와 코드 내용

프롬프트 히스토리는 이 기능과 분리된 기존 opt-in 수집 경로다. 도구 메타데이터를 켜도 프롬프트 수집이 자동으로 켜지지 않는다.

## 감지 방식과 한계

- Claude Code의 `Skill` 도구 호출은 `explicit`로 기록한다.
- Codex는 알려진 스킬 루트의 `SKILL.md` 로드를 `derived_load`로 기록한다. 독립적인 스킬 호출 이벤트가 없는 경우가 있으므로 화면에는 “로드”로 구분한다.
- Cursor는 `~/.cursor/projects/**/agent-transcripts`의 JSONL·레거시 TXT에서 명시적으로 식별되는 MCP·Skill 호출만 `explicit`로 기록한다. Cursor가 도구 결과를 트랜스크립트에 남기지 않은 호출은 성공으로 추측하지 않고 `unknown`으로 둔다.
- 플러그인 키는 `superpowers:brainstorming` 같은 명시적 접두사나 Codex 플러그인 캐시 경로처럼 출처가 확인되는 경우에만 기록한다. 추측으로 연결하지 않는다.
- MCP 도구명은 서버와 도구 이름만 정규화한다. 호출 인자는 읽더라도 활동 이벤트에 보관하거나 직렬화하지 않는다.
- Cursor 인벤토리는 `~/.cursor/mcp.json`, `~/.cursor/skills`, `~/.cursor/skills-cursor`, 공유 `~/.agents/skills`에서 이름만 읽는다. MCP 명령·인자·환경변수와 `SKILL.md` 본문은 전송하지 않는다.

## 부하 제어

- 사용량과 도구 활동은 같은 로그 읽기와 JSON 파싱 패스를 공유한다.
- 도구 활동은 기능을 처음 켠 시점 이전 로그를 백필하지 않고 이후 변경분부터 수집한다.
- 인벤토리는 설정 경로의 변경 stamp를 먼저 확인하고, 변경이 없으면 전체 스캔을 최대 24시간에 한 번만 수행한다.
- 동일한 fingerprint는 다시 전송하지 않는다.
- 서버가 새 엔드포인트에 `404` 또는 `405`를 반환하면 24시간 동안 재시도를 미뤄 구버전 서버에 부하를 주지 않는다.
- 릴리스 벤치마크는 대표 Codex 픽스처에서 기존 사용량 파서 대비 CPU 증가가 10% 이내인지 검사한다.

## 화면과 개인정보 경계

- 개인 화면은 본인의 도구별 활동과 기기별 인벤토리를 보여준다.
- 조직 화면은 사용자 이름, 기기명, 세션, 도구 이름을 노출하지 않고 종류별 익명 합계만 보여준다.
- ingest 토큰 소유권으로 개인 데이터를 귀속하며, 요청이 주장한 사용자 ID는 신뢰하지 않는다.

## 비활성화

설치할 때 도구 메타데이터 수집을 끄려면 다음처럼 실행한다.

```bash
curl -fsSL <toard 주소>/install.sh | TOARD_INGEST_TOKEN=<내 토큰> TOARD_SHIM_COLLECT_TOOLS=0 sh
```

이미 설치했다면 `~/.toard/credentials`에 아래 값을 추가하거나 변경한다.

```text
collect_tools=false
```

일시적으로 끌 때는 실행 환경에 `TOARD_SHIM_COLLECT_TOOLS=0`을 지정할 수도 있다. `0`, `false`, `off`, `no`를 비활성 값으로 인식한다.
